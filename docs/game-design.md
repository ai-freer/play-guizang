# 《消消藏师傅》基础设计文档

## 1. 项目定位

《消消藏师傅》是一款基于 25 格真人 meme 表情素材的网页版休闲三消游戏。核心卖点不是复杂规则，而是“把同一个人的不同表情做成可消除的梗图棋子”，形成轻量、好笑、适合截图传播的娱乐体验。

目标形态：
- 平台：桌面与移动浏览器。
- 类型：经典 match-3 / 消消乐。
- 节奏：单局 1-3 分钟。
- 操作：鼠标拖拽或点击交换；移动端触摸滑动或点选交换。
- 视觉：综艺感、夸张、亮面伪 3D、弹幕梗图气质。

## 2. 核心素材

素材来源：
- 以 `5x5` meme 矩阵作为基础素材库。
- 共 25 个表情图标，每个图标保留原字幕。
- 每个表情作为一种棋子类型。

推荐资源处理：
- 从 25 宫格原图裁切出 `25` 张独立 tile。
- 每张 tile 输出为统一尺寸，例如 `256x256` PNG 或 WebP。
- 游戏中再叠加统一的伪 3D 外壳、描边、阴影、选中态，而不是把外壳烘焙进素材。

建议资源命名：
- `emoji_01_a.png`
- `emoji_02_really.png`
- `emoji_03_served.png`
- 也可在 manifest 中维护中文名，不把中文直接放进文件名。

## 3. 基础玩法

棋盘：
- 默认 `7x7`。
- 后期可扩展到 `8x8`，但首版建议先固定 `7x7`。

玩家操作：
- 选择相邻两个棋子交换。
- 若交换后形成横向或纵向 `3` 个及以上同类棋子，则交换成立。
- 若没有形成消除，则棋子回弹到原位。

消除规则：
- `3连`：普通消除。
- `4连`：生成一个横向或纵向爆破道具。
- `5连`：生成彩虹 meme，道具可消除某一类表情。
- `T/L 型`：生成炸弹道具，影响周围范围。

回合流程：
1. 玩家交换棋子。
2. 判定是否形成匹配。
3. 消除匹配棋子并播放动画。
4. 上方棋子下落填补空位。
5. 顶部生成新棋子。
6. 检查是否形成连锁消除。
7. 更新目标、分数、步数。
8. 若目标完成则过关；若步数耗尽仍未完成则失败。

## 4. 目标与计分

每关目标采用 1-3 个任务组合：
- 收集指定表情：例如消除 `啊？ x8`。
- 达到分数：例如 `目标分 12,000`。
- 清除障碍：后续可加入“冰块”“锁链”“马赛克遮罩”等。

基础计分：
- 普通 3 连：`60` 分。
- 4 连：`120` 分，并生成特殊棋子。
- 5 连：`240` 分，并生成彩虹 meme。
- 连锁倍率：每次连续下落消除，倍率 `+0.5`。
- 例：第一轮 `1x`，第二轮 `1.5x`，第三轮 `2x`。

分数反馈：
- 消除位置弹出 `+60`、`+120`、`连消 x4`。
- 重要连锁使用更大的综艺式红黄标签。

## 5. 道具设计

首版保留 4 个道具，但 UI 需要显著升级为“游戏按钮”而不是扁平卡片。

### 5.1 换位

功能：
- 允许玩家交换任意两个棋子。
- 不要求交换后能消除。

UI 造型：
- 蓝色圆形核心，双向箭头符号。
- 外层为厚实奶油色底座。
- 底部有深色投影，顶部有高光。

### 5.2 炸弹

功能：
- 点击一个棋子，清除周围 `3x3` 范围。

UI 造型：
- 红色核心，爆裂星形符号。
- 外框可使用危险感红边。
- 点击时按钮短暂放大并闪白。

### 5.3 刷新

功能：
- 重新洗牌棋盘，不减少目标进度。
- 可消耗一步或不消耗，首版建议消耗一步，避免太强。

UI 造型：
- 绿色核心，旋转箭头。
- 按钮周围有环形流动高光。

### 5.4 提示

功能：
- 高亮一个可消除交换。

UI 造型：
- 金色核心，问号或灯泡符号。
- 提示目标棋子外圈发光并轻微跳动。

### 5.5 道具 UI 规格

单个道具按钮结构：
- 背板：圆角矩形，厚底座，奶油色或浅金色。
- 核心：圆形彩色按钮，内嵌图标。
- 伪 3D：上高光、下阴影、内阴影、底部偏移暗面。
- 数量角标：右上红色圆形徽章。
- 状态：
  - 默认：明亮、可点击。
  - 按下：整体下移 `3-5px`，底部阴影收窄。
  - 禁用：降低饱和度和透明度。
  - 选中：外圈发光，棋盘进入目标选择模式。

## 6. 视觉方向

关键词：
- 综艺字幕
- 亮面糖果
- 伪 3D 按钮
- 街机休闲
- 表情包梗图

色彩：
- 背景：深青绿 / 深蓝绿色舞台感。
- 主强调：荧光粉红、金黄、青绿、亮蓝。
- 内容底板：奶油白，避免纯白原型感。
- 棋盘：深色外框 + 高亮描边 + 浅色格子。

字体：
- 中文 UI 建议使用系统可用的粗体中文：PingFang SC / 思源黑体 / 阿里巴巴普惠体等。
- 游戏标题和弹窗可以使用更重的描边字。

棋子视觉：
- 表情 tile 不应裸露摆放。
- 每个 tile 放入统一伪 3D 外壳：
  - 圆角方块。
  - 下方深阴影。
  - 顶部白色高光。
  - 外圈依据类型轻微变色。

动画方向：
- 交换：快速滑动，带轻微 overshoot。
- 无效交换：交换后回弹。
- 消除：放大、闪白、粒子爆开。
- 下落：重力感下落，落地轻微弹跳。
- 连锁：屏幕轻微震动，弹出倍率标签。

## 7. 关卡难度设计

难度核心不是复杂机关，而是逐渐增加“棋子种类数”和“目标压力”。

### 7.1 难度参数

可调参数：
- `boardSize`：棋盘大小。
- `tileTypes`：本关出现的表情种类数。
- `moveLimit`：步数。
- `targets`：目标数量和目标类型。
- `specialSpawnRate`：特殊棋子出现机会。
- `obstacles`：障碍格数量。

首版建议只使用：
- 棋子种类数
- 步数
- 收集目标
- 分数目标

障碍物放到第二阶段。

### 7.2 关卡递增方案

第 1-3 关：新手
- 棋盘：`7x7`
- 表情种类：`5-6`
- 目标：单一收集目标
- 步数：`18-22`
- 目的：让玩家理解交换、消除、下落。

第 4-7 关：基础挑战
- 表情种类：`7-9`
- 目标：两个收集目标，或一个收集目标 + 分数目标。
- 步数：`16-20`
- 引入：4 连特殊棋子。

第 8-12 关：连锁体验
- 表情种类：`10-13`
- 目标：两个收集目标 + 分数目标。
- 步数：`16-18`
- 引入：5 连彩虹 meme。

第 13-18 关：中等难度
- 表情种类：`14-18`
- 目标：三个收集目标。
- 步数：`15-18`
- 要求玩家开始主动制造特殊棋子。

第 19-25 关：完整素材池
- 表情种类：`19-25`
- 目标：三个目标，数量更高。
- 步数：`14-17`
- 可加入轻量障碍，例如固定冰块。

## 8. 系统架构建议

推荐技术栈：
- Phaser 3
- TypeScript
- Vite
- DOM HUD + Canvas 棋盘

核心原则：
- 游戏规则状态独立于 Phaser Scene。
- Phaser 只负责渲染、动画、粒子、输入转发。
- HUD 使用 DOM/CSS，方便响应式和中文排版。

推荐目录：

```text
src/
  main.ts
  game/
    Match3Game.ts
    Board.ts
    Rules.ts
    LevelConfig.ts
    Scoring.ts
    Shuffle.ts
  scenes/
    BootScene.ts
    PreloadScene.ts
    GameScene.ts
  ui/
    Hud.ts
    ToolButton.ts
    overlays.ts
  assets/
    manifest.ts
  data/
    levels.ts
  styles/
    theme.css
public/
  assets/
    tiles/
    ui/
docs/
  game-design.md
```

## 9. 数据模型

棋子：

```ts
type Tile = {
  id: string;
  type: number;
  special?: "row" | "column" | "bomb" | "rainbow";
};
```

棋盘：

```ts
type BoardState = {
  width: number;
  height: number;
  cells: Array<Tile | null>;
};
```

关卡：

```ts
type LevelConfig = {
  id: number;
  boardSize: { width: number; height: number };
  tileTypes: number[];
  moveLimit: number;
  targets: Array<
    | { kind: "collect"; type: number; count: number }
    | { kind: "score"; score: number }
  >;
};
```

## 10. 首版 MVP

MVP 必须包含：
- 25 个 meme tile 裁切与加载。
- 7x7 棋盘。
- 基础交换。
- 匹配检测。
- 消除、下落、补新。
- 连锁消除。
- 分数与步数。
- 关卡目标。
- 失败与胜利弹窗。
- 4 个道具的 UI 外观，至少实现“提示”和“刷新”两个功能。

首版可以暂缓：
- 登录、排行榜、存档。
- 音频。
- 复杂障碍。
- 多章节地图。
